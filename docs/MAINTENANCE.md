# Как вести документацию

Руководство по формату и поддержке документации проекта.

---

## Структура документации

```
docs/
├── README.md         # Оглавление, навигация
├── ARCHITECTURE.md   # Архитектура, стек, структура
├── SETUP.md          # Установка, переменные окружения
├── API.md            # REST API
└── MAINTENANCE.md    # Этот файл
```

---

## Формат файлов

- **Markdown (.md)** — стандарт для документации в репозиториях
- Кодировка: UTF-8
- Переносы строк: LF (Unix)

---

## Что где описывать

| Документ | Содержимое | Когда обновлять |
|----------|------------|-----------------|
| **README.md** (корень) | Краткое описание, быстрый старт, ссылки на docs | При изменении структуры проекта |
| **docs/README.md** | Оглавление, ссылки на разделы | При добавлении нового раздела |
| **ARCHITECTURE.md** | Стек, структура папок, модель данных, CI/CD | Новый модуль, изменение архитектуры |
| **SETUP.md** | Шаги установки, переменные окружения | Новые зависимости, env-переменные |
| **API.md** | Эндпоинты, request/response, коды ошибок | Новый или изменённый API |

---

## Правила оформления

1. **Заголовки** — `#` для H1, `##` для H2 и т.д. Один H1 на файл.
2. **Таблицы** — для списков с колонками (метод, путь, описание).
3. **Блоки кода** — с указанием языка: ` ```json `, ` ```bash `.
4. **Ссылки** — относительные: `[API](API.md)`.
5. **Примеры** — реальные значения, не плейсхолдеры типа `your-value`.

---

## Примеры хорошей документации

- [NestJS Docs](https://docs.nestjs.com) — структура, примеры кода
- [Prisma Docs](https://www.prisma.io/docs) — API reference
- [GitHub Docs](https://docs.github.com) — пошаговые инструкции

---

## Чеклист при добавлении фичи

- [ ] Обновить ARCHITECTURE.md (модуль, модель данных)
- [ ] Обновить API.md (новые эндпоинты)
- [ ] Обновить SETUP.md (если нужны новые env-переменные)
- [ ] Добавить примеры в API.md (request/response)
